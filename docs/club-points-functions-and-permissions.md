# 俱乐部员工积分系统功能与权限规格

## 角色与数据范围

第一版三类线上角色：

| 角色 | 是什么 | 基本范围 |
| --- | --- | --- |
| 员工 | 企业员工、俱乐部参与者 | 只能操作本人数据和已加入俱乐部的可见数据 |
| 俱乐部负责人 | 拥有额外管理能力的员工 | **员工全部能力** + 管理自己负责的俱乐部 |
| 系统管理员 | 全局配置、审核、处理、台账、审计 | 全局 |

负责人不是独立用户体系，是“员工 + 负责俱乐部关系”。凡是员工能做的本人操作，负责人作为员工同样能做。

数据范围三档：

| 范围 | 含义 |
| --- | --- |
| 本人 | 只能操作/查看自己的记录 |
| 已加入俱乐部 | 员工对自己已加入的俱乐部可见（成员名单、已发布活动）|
| 负责俱乐部 | 负责人对自己负责的俱乐部可管理 |
| 全局 | 管理员对全部数据可见可操作 |

权限判断必须两层：先判角色能不能做这个动作，再判这个动作是否落在允许的数据范围内。两层都在后端做，前端隐藏按钮不算权限。

## 功能清单

系统的全部功能动作，按功能域分组。这是“系统能干哪些事”的功能视角，不是模块依赖图。

**俱乐部与成员**：浏览可加入俱乐部、加入俱乐部、退出俱乐部、查看成员名单、创建俱乐部、修改俱乐部信息、停用俱乐部、物理删除俱乐部、设置/移除负责人、移除成员。

**活动**：查看已发布活动、创建活动草稿、修改活动、提交发布审核、撤回待审活动、审核活动发布、直接发布活动、修改已发布活动（普通信息）、修改已发布活动（关键信息）、取消活动、物理删除活动。

**报名与签到**：报名活动、自助取消报名、签到、签退、补录/修正签到签退、标记特殊缺席、手动触发活动结算。

**积分账本**：查看本人积分概览与流水与来源统计、查看自己负责俱乐部发放来源的积分流水与成员摘要、管理员查看全局积分账户与流水、手工调整积分、撤销积分流水。

**非签到类积分**：提交非签到材料、撤回/修改材料（审核前）、物理删除材料、审核非签到材料、管理员代录积分、管理员违规扣分、管理员弄虚作假处理。

**兑换**：查看开放批次与礼品、提交兑换申请、取消兑换申请（审核前）、查看本人兑换记录、维护兑换批次（建/改/开/关）、维护礼品与库存、审核兑换申请。

**规则版本**：创建/修改规则版本、发布规则版本、撤回/停用规则版本。

**异议**：提交异议、查看本人异议、处理异议。

**年度与预算**：年度清零（手动触发）、查看俱乐部年度排名、生成激励建议、维护预算记录。

**报表与审计**：导出报表、查看审计日志、查看/处理业务任务。

**通知与待办**：查看本人通知与标记已读、负责人待办、管理员待办。

## 关键范围规则

- 员工只能操作本人报名、签到、签退、兑换、异议；只能查看已加入俱乐部的成员名单和已发布活动。
- 员工退出俱乐部后，仍能查看自己参与过的历史活动和积分流水，但看不到该俱乐部新活动和成员名单。
- 负责人只能管理自己负责的俱乐部、活动、报名、签到、材料；只要积分来源是自己负责俱乐部分发，就可查看对应积分流水与成员积分摘要，这里的“其他/别人”只指其他俱乐部、无俱乐部来源或全局来源，不包括自己负责俱乐部分发给成员的积分；**不能**查看员工在其他俱乐部、无俱乐部来源或全局来源下的积分明细，**不能**查看员工全量账户与全量兑换记录，**不能**审核（含本俱乐部材料），**不能**导出报表，**不能**设置/移除负责人。
- 负责人可以查看已退出成员在本俱乐部历史活动中的参与、签到和积分材料。
- 管理员拥有全局范围；积分调整、代录、撤销等敏感积分操作仍需原因或材料，并写审计。

## 强确认与强审计清单

**强确认**：仅物理删除俱乐部需要强确认。

**强审计**（必须与业务写在同一事务，审计失败则业务失败）：修改已发布活动关键信息、取消活动、物理删除（俱乐部/活动/材料）、补录或修正签到签退、管理员代录积分、违规扣分、弄虚作假处理、审核非签到材料、审核兑换、手工调整或撤销流水、修改兑换资格规则、年度清零人工处理、报表导出、设置/移除负责人、移除成员、停用/删除俱乐部、发布/停用规则版本、处理异议。

审计日志只允许系统管理员查看，暂定保留三年。强确认不等于审核。
