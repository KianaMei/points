# 项目范围

## 目标

- 做俱乐部员工积分系统。
- 三类线上角色：员工、俱乐部负责人、系统管理员。
- 负责人不是独立用户体系，是“员工 + 负责俱乐部关系”。
- 积分结果必须可追溯、可审计、可从流水重算。

## MVP 闭环

```text
俱乐部和成员
-> 活动创建、报名、签到签退
-> 活动结算生成积分流水
-> 员工查看余额、流水、来源统计
-> 兑换申请、冻结、库存锁、审核扣减
-> 俱乐部年度排名和激励建议
-> 年度清零
-> 报表、审计、任务追踪
```

## 第一版做

- 俱乐部、成员、负责人。
- 活动、报名、签到、签退、修正、特殊缺席。
- 活动自动结算和缺席扣分。
- 负责人提交非签到材料，管理员审核。
- 管理员代录、调整、撤销积分。
- 兑换批次、资格快照、礼品库存、兑换审核。
- 规则版本和规则项配置。
- 预算记录、年度清零、排名、激励建议。
- 系统内通知、待办、报表、强审计。

## 第一版不做

- 积分来源消费分摊。
- FIFO、LIFO、按比例扣减来源积分。
- 员工自助补登记。
- 领取、签收、逾期未领取、自动放弃。
- 礼品质量问题线上闭环。
- 经费审批流、领导小组线上审批流。
- 外部组织架构、企业微信、钉钉、短信、邮件强依赖。
