# 权限、范围、审计

## 角色

- 员工：本人数据和已加入俱乐部可见数据。
- 俱乐部负责人：员工能力 + 自己负责俱乐部管理能力。
- 系统管理员：全局管理、审核、报表、审计。

## 数据范围

- 本人。
- 已加入俱乐部。
- 负责俱乐部。
- 全局。

## 权限原则

- 先判断动作权限。
- 再判断数据范围。
- 两层都在后端做。
- 前端隐藏按钮不算权限。

## 权限码

- 统一格式：`clubpoints:<resource>:<action>`。
- Controller 使用 `@PreAuthorize`。
- Service 使用 `ClubScopeService` 做数据范围。

## 负责人限制

- 不能看全局兑换。
- 不能看其他俱乐部积分明细。
- 不能审核非签到材料。
- 不能导出报表。
- 不能设置或移除负责人。

## 强确认

- 仅物理删除俱乐部需要强确认。
- 强确认不等于审核。
- 强确认不替代权限和审计。

## 强审计

这些动作必须写强审计：

- 修改已发布活动关键信息。
- 取消活动。
- 物理删除俱乐部、活动、材料。
- 补录或修正签到签退。
- 管理员代录、调整、撤销积分。
- 审核非签到材料、审核兑换。
- 修改兑换资格规则。
- 年度清零人工处理。
- 报表导出。
- 设置或移除负责人、移除成员。
- 发布或停用规则版本。
- 处理异议。

强审计失败，业务必须回滚。
