# 数据流和事实源

## 总原则

- `club_points_transaction` 是唯一积分事实源。
- 余额缓存不是事实源。
- 冻结不是流水。
- 报表不是主表。
- 导出动作写强审计，不新增导出业务表。

## 活动发分

```text
活动发布 -> 员工报名 -> 签到 / 签退
-> 结算读取报名、签到、特殊缺席、活动积分配置
-> 生成活动积分或缺席扣分流水 -> 更新账户缓存
```

幂等靠 `club_points_transaction.idempotency_key`。

## 非签到积分

```text
负责人提交材料 -> 管理员审核
-> 审核通过锁定材料和附件
-> 每条材料明细生成积分流水 -> 通知员工
```

## 兑换

```text
批次开启生成资格快照 -> 员工提交申请
-> 同事务冻结积分 + 锁库存 + 创建申请 -> 管理员审核
-> 通过：冻结转扣减，生成兑换负向流水，库存转已用
-> 拒绝/取消：释放冻结和库存，不生成扣减流水
```

## 年度清零

```text
每年 1 月 1 日 00:00 北京时间
-> 清未冻结可用积分 -> 生成年度清零负向流水
-> 不删除历史流水
```

跨年冻结兑换被拒绝时释放回账户，不追加过期清零。

## 异议和调整

- 异议不直接改积分。
- 需要改积分时走调整流水或撤销流水。
- 原流水不删除，撤销用反向流水。

## 报表

- 积分明细、总台账、兑换记录、俱乐部排名都从流水和业务快照查询。
- 员工和负责人不导出。
- 管理员导出必须写强审计。
